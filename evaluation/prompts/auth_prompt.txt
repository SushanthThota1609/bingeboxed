=== FUNCTIONAL REQUIREMENTS ===
FR-01: POST /api/auth/register
  - Accepts { email, password }
  - Validates email format
  - Password minimum 8 characters
  - Returns 201 on success
  - Returns 400 on validation failure
  - Returns 409 if email already exists

FR-02: POST /api/auth/login
  - Accepts { email, password }
  - Returns 200 with { token } on success
  - Returns 401 on invalid credentials

FR-03: JWT tokens expire after 24 hours

FR-04: All endpoints except /api/auth/** require
       valid JWT in Authorization: Bearer header

FR-05: Invalid or expired token returns 401

FR-06: GET /login → renders templates/auth/login.html
FR-07: GET /register → renders templates/auth/register.html

=== SECURITY REQUIREMENTS ===
SEC-01: No sensitive data in JWT payload beyond email
SEC-02: No credentials or secrets hardcoded anywhere
SEC-03: Passwords never logged or returned in responses
SEC-04: SQL injection prevention via JPA parameterized queries
